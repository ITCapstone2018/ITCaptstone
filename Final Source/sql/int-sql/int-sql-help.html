Interactive SQL Sub-module

This reference is intended to show you how to use the various topics of the "Interactive SQL" Sub-module. Choose a topic from below to be directed to its associated help.

DDL and DML
Creating a Table 

First, create a table by clicking the yellow "Create Table" button.

Once you have created a table, information regarding the created table will be placed in the Information window. Additionally, the SQL code used to create the table will be shown underneath the information, and a visual of the created table will be shown to the right of the information. Because you've already created a table, the "Create Table" button will now be grayed out and unusable.

Modifying Tables with DDL

Now, you can use DDL commands to modify the table that you have just created. First, use the highlighted "Alter Table" button to change the table.

After clicking the "Alter Table" button, information on the process and the related SQL code will be displayed on screen. Moreover, the PET table will visually change to represent the effect of the alter command with the addition of a "Breed" column. After you have read the information on screen, you can proceed to the next highlighted button, "FK Constraint".

Modifying Tables with DML

To practice modifying a table with DML, first click on the DML tab at the top of the module. Then, to start the module, click the highlighted "Insert" button.

Continue clicking the highlighted buttons to proceed through the DML exercise. If at anytime in the module you want to start over, you can click the "Reset" button to revert back to the beginning.

Queries - Basic and Advanced
Select a Query Type

Start by choosing a query type from the Scenario's tabbed navigation bar at top with each tab representing a different type. By default, the first query type of each scenario will be opened, or Projection for Queries - Basic and Multiplication for Queries - Advanced.

Perfrom a Query

Perform a query by filling in all required fields by clicking the downward arrow and selecting a value from the list.

Select a View Type

Start by choosing a view type from the Scenario's tabbed navigation bar at top with each tab representing a different type of view. By default, the content on Regular Views will be displayed.

Create a View

Create a view by filling in all required fields of the "SQL Command" window by clicking each downward arrow and selecting a value from the list. For the "Create View" field, you can type a view name of your choice, or click the downward arrow to select one of the sample view names. Note that not allof the fields are required.

Results

When all required fields of the "SQL Command" window have been filled, the program will automatically generate a view with the name you entered, and it will show the SQL Query Syntax that would be used to create it.

To explore the other possibilities for the created view, change one or more fields in the "SQL Command" window. The generated view and the generated "SQL Query Syntax" will update instantly with each change. Conversely, you can use the "Save View" button at the top of the screen to save your view. Saved views can be accessed from the "Saved Views" table on the right.

Misconceptions
Select a Misconception

Start by choosing one of the Misconceptions from the Scenario's tabbed navigation bar at top with each tab representing a different Misconception. By default, the content on the first Misconception will be displayed.

View the Effect of Incorrect SQL

The scenario will start by showing what will happen if the SQL is written incorrectly, shown in the "Incorrect SQL Statement" window. The objective at hand is shown in the top left "Objective: Incorrect Module" window. To navigate through the scenario, click on the "Next Step" button at the top of the screen. Continue pressing "Next Step", paying close attention to the temporary tables and error windows, to work all the way to the end of the scenario. Once the example is finished, the "Reset" button will start it over and the "Correct" button will show how the SQL should have been written.

View the Correct SQL

Clicking the "Correct" button will change the scenario to show how to correctly meet the objective. Once the "Correct" button has been clicked, the background color will shift from red to green and the Correct example will be displayed. The navigation of the correct versions of the Misconceptions works very similarly to the incorrect versions by moving through the scenario with the "Next Step" button and starting over with the "Reset" button.
