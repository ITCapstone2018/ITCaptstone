Interactive SQL

SQL

SQL is an acronym that stands for structured query language. SQL is not a programming language, but rather a data sub-language that allows users to interact with a database. As a sub-language, SQL supports two primary functions data definition and data manipulation. 




DML and DDL

SQL is comprised of two languages, Data Definition Language (DDL) and Data Manipulation Language (DML) to accomplish the two basic functions of SQL. The Data Definition Language (DDL) is used to define the tables and structures of a database. CREATE, ALTER, and DROP are all examples of DDL in SQL. The Data Manipulation Language (DML) is used to specify, retrieve, and modify the data in a database. SELECT, INSERT, UPDATE, and DELETE are examples of DML in SQL.

Basic SQL Queries

Projection, or the SELECT statement, selects the data of a table that will be displayed. Restriction, or the WHERE clause, selects only the data of a table that meets certain criteria. Functions, such as COUNT, SUM, and AVG, allow a value to be calculated from other values in the table. The GROUP BY clause will group rows by common values. The HAVING clause is used to determine which groups the GROUP BY is to include. The DISTINCT keyword will eliminate duplicate rows in the result table. ORDER BY will reorder the results in by descending or ascending order.

Advanced SQL Queries

Multiplication displays all possible combinations of two tables. Joins create a new table by connecting two other tables. Outer joins will create a new table that connects two other tables while also displaying the rows of one table that have no matching values from the other. In other words, outer joins will result in null values. A union combines two tables that have identical attributes. An intersection will display all rows shared in common by two tables. A difference displays all rows that exist in one table but not the other.  

  Views

   A view is essentially a virtual table based on the results of a SQL SELECT            statement. Most views, however, are not stored as tables but as objects that        retain their related SQL statement. A materialized view, on the other hand, is      actually stored as a table in the database though it appears the same as a          regular view to the end user. By storing as a table, a materialized view speeds    up query results because there is no SQL statement to be executed.                  Conversely, materialized views slow down updates because every change to      the database requires the materialized view table to be updated. View syntax    in SQL looks like this: CREATE VIEW view_name AS SELECT column_name(s),      FROM table_name, WHERE condition

The first misconception             involves using two tables without an appropriate join clause. This error results       in the multiplication of tables rather than correctly creating a relationship.            Multiplication of tables will display all possible combinations of data whereas a       joining of tables will only show the specified, logical combinations of data. 2.         The second misconception is how to use 'AND' and 'OR' correctly.  AND will       limit the results displayed.  OR will extend the results displayed. 3. The final         misconception involves confusing the function oriented NOT EQUALS with the       variable oriented NOT IN.
