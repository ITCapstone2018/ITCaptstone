Referential Integrity Sub-module

This reference is intended to show you how to use the "Referential Integrity" Sub-module.

Setup Table Conditions  

The Referential Integrity Sub-module will practice keeping consistency between two tables. The "Table Setup SQL" will allow the table conditions to be changed. By default, "RESTRICT" is selected for both on update and on delete.

Change the Data

After the table conditions have been setup, the "SQL Commands" window can be used to update and delete data. Use the drop down fields to select the conditions.

Perform an Update or Delete

After setting up both the table conditions and editing the SQL update/delete commands, use the "Update" and "Delete" buttons to perform the operation on the table. Pressing a button once will show highlights illustrating what has happened in the tables and SQL commands. The button will change to say "Continue" which, when clicked, will display a window describing if the performed operation meets the rules of Referential Integrity.

At the end of the example, the "Reset" button can be used to restore the scenario. Change the conditions and explore the possibilities and unique combinations.
