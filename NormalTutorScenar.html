Learn about the basics of normalization and the various forms associated with it.

What is Normalization?

Normalization is the process of assigning attributes to entities. It reduces data redundancy and helps reduce anomalies associated with redundancy. A table can be normalized by going through a series of stages called normal forms.

UnNormalized

The table to the right represents unnormalized data. This can be indicated by (1) an intended primary key including null values; (2) multiple values in a column/row intersection; or (3) anomalies occuring during updates, or deletions. For example, if professor Harley changes his office, the room number would have to be changed in each row it occurs in.

First Normal Form

First normal form involves (1) defining the key attributes; (2) eliminating null values in those key attributes; and (3) confirming that all other non-key attributes are dependent on the primary key. In this case, both Course ID and SSN are defines as the composite primary key because that combination uniquely defines every row.

Second Normal Form

Second normal form involves removing partial dependencies, or attributes that are dependent on only part of the composite key. To do this, break the original table into seperate tables. Each part of the composite key may become the primary key in its own table. Another table may be created, if necessary, whose key remains the composite key. Then add the remaining attributes to the new tables corresponsing to the part of the third key it is dependent upon. Note that the Student and Grade tables are automatically in third normal form at this point.

Third Normal Form

Third normal form involves removing transitive dependencies. A transitive dependency occurs when a non-key attribute is dependent on another non-key attribute. In this case, the floor is dependent on the room number. Since both are non-key attributes, a seperate table is created with those attributes in them, taking note that one attribute remains in the 2NF table to link the new table with it using a foreign key
